# Регрессионный чеклист: созвоны

## 1. Проверка деплоя
- Убедиться, что production/preview стенд собран из актуального коммита ветки `dev`.
- Проверить, что версия UI в браузере соответствует последнему коммиту (hard reload + без кеша).

## 2. Проверка создания и видимости в админке
- Выбрать в верхнем селекторе конкретный поток.
- Создать созвон с опцией `Все потоки`.
- Убедиться, что после сохранения созвон остаётся в списке (в режиме потока показываются и общие).
- Переключить селектор на `Все потоки` и проверить, что отображаются все записи.

## 3. Проверка видимости на дашборде
- Перейти в student mode и открыть дашборд.
- Убедиться, что для будущего созвона отображается карточка "Следующий созвон".

## 4. Проверка ошибок загрузки
- При искусственной ошибке API `/api/calls/upcoming` проверить наличие ошибки в консоли:
  `Error fetching upcoming call: ...`

## 5. Проверка факта записи в БД
- Выполнить запрос в БД: `use vibes-database`

```sql
SELECT id, topic, cohort_id, scheduled_date, scheduled_time, created_at
FROM admin_calls
WHERE deleted_at IS NULL
ORDER BY created_at DESC
LIMIT 10;
```
